# Руководство контрибьютора

Этот репозиторий содержит контракты Protobuf и автогенерацию артефактов. Все изменения должны быть воспроизводимыми и проходить lint.

## Основные правила

1. Источник истины — файлы в папке [proto](proto).
2. Сгенерированный код в [gen](gen) не редактируется вручную.
3. Любые изменения .proto должны сопровождаться обновлением документации и автогенерации.
4. Соблюдайте обратную совместимость (buf breaking). Несовместимые изменения допускаются только с явным обоснованием.

## Процесс изменений

1. Создайте ветку.
2. Измените или добавьте .proto файлы в [proto](proto).
3. Запустите генерацию:
   - make generate
4. Запустите проверку:
   - make lint
5. Убедитесь, что изменённые сервисные папки содержат README.md.
6. Оформите PR с понятным описанием изменений и их влияния на контракт.

## Нейминг и версия

- Каждый сервис хранится в отдельной папке (например, [proto/auth](proto/auth)).
- Версии — в подпапках (например, [proto/auth/v1](proto/auth/v1)).
- Изменения должны соответствовать SemVer на уровне сервиса.

## Документация

- HTML документация хранится в [docs](docs).
- OpenAPI спецификации — в [docs](docs) и [gen/openapiv2](gen/openapiv2).
